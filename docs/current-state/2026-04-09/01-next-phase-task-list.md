## 1. 目标

下一阶段不再继续堆“页面壳子”，而是把现有 App 从“可演示”推进到“可联调、可走主链、可继续扩开发”的状态。

本阶段总目标分成 4 条:

1. 把广场、圈子从本地静态数据切到真实后端
2. 把账号中心里的设置、通知、设备、黑名单等后端能力真正接进移动端
3. 把聊天里的占位媒体能力升级成真实上传链路
4. 把当前后端已有但前端未接的主流程补齐

## 2. 推荐执行顺序

建议按这个顺序往下做:

1. 广场前端接真实 API
2. 圈子前端接真实 API
3. 设置 / 通知 / 设备 / 黑名单 / 注销接通
4. 聊天真实媒体上传
5. 密码重置页面
6. 社会化登录真实接入
7. 后台前端

原因:

- 前 3 项是“现有后端已具备、移动端最缺”的部分，投入产出比最高
- 第 4 项会带动聊天和圈子的真实内容能力
- 第 5 到第 7 项更适合在前面主链稳定后再做

## 4. 任务详情

### T01 广场接真实 API

目标:

- 把广场页从本地 `_profiles` 和本地通知，切换到真实后端

涉及能力:

- `GET /api/v1/square/banner`
- `GET /api/v1/square/notices`
- `GET /api/v1/square/users`

前端完成定义:

1. 页面启动后拉取 Banner
2. 页面启动后拉取系统通知
3. 页面启动后拉取推荐用户
4. 切换地区 / 性别等筛选时重新请求用户列表
5. 加载中、空状态、错误状态都可见

验收标准:

1. 断网时能看到错误提示
2. 有数据时展示真实后端返回
3. 参数变化时返回结果同步变化

依赖:

- 无新增后端依赖

### T02 广场用户详情与搜索

目标:

- 给广场加“看人”的详情页和搜索页

涉及能力:

- `GET /api/v1/users/:userId`
- `GET /api/v1/search/users?q=关键词`

前端完成定义:

1. 点用户卡进入详情页
2. 详情页展示基础资料、视频介绍、作品
3. 提供搜索入口
4. 搜索结果列表可复用用户卡

验收标准:

1. 广场卡片到详情页跳转完整
2. 搜索空结果可见
3. 搜索结果能再次进入详情页

### T03 广场筛选条件保存

目标:

- 把广场筛选做成可保存、可恢复

涉及能力:

- `POST /api/v1/square/filters`
- `GET /api/v1/square/filters`

前端完成定义:

1. 增加“保存当前筛选”
2. 增加“已保存筛选”列表
3. 点选后恢复到页面筛选器

验收标准:

1. 保存后重进页面仍能取到历史筛选
2. 恢复后能重新拉取正确用户列表

### T04 圈子列表和发布接真实 API

目标:

- 让圈子从本地 `_posts` 切到真实后端

涉及能力:

- `GET /api/v1/circle/posts`
- `POST /api/v1/circle/posts`
- `PUT /api/v1/circle/posts/:postId`

前端完成定义:

1. 列表页从接口加载动态
2. 发布页提交真实接口
3. 编辑动态时走更新接口
4. 发布成功后刷新真实列表

验收标准:

1. 发布的新动态刷新后仍存在
2. 编辑内容刷新后仍存在
3. 草稿机制保留

依赖:

- 最好与 T06 一起排期，这样素材不会继续是假数据

### T05 圈子详情、评论、举报

目标:

- 把圈子从“只有流”补成“有详情、有互动”

涉及能力:

- `POST /api/v1/circle/posts/:postId/reports`

前端完成定义:

1. 动态详情页
2. 评论列表
3. 评论输入
4. 回复评论
5. 举报入口

验收标准:

1. 列表进入详情
2. 发表评论后数量刷新
3. 举报后有成功反馈

### T06 圈子发布页接真实上传

目标:

- 把圈子发布里的图片、语音、作品从“模拟选择”升级为真实文件链路

涉及能力:

- `POST /api/v1/upload/image` 不存在
  当前应使用:
  - `POST /api/v1/upload/single`
  - `POST /api/v1/upload/audio`
  - `POST /api/v1/upload/video`

前端完成定义:

1. 选择图片后真实上传
2. 选择语音后真实上传
3. 作品仍可来自本地作品库
4. 发布时把真实媒体 URL 带给圈子接口

验收标准:

1. 上传成功后服务端返回 URL
2. 发布后的动态详情能看到对应附件元数据

### T07 设置页真实持久化

验收标准:

1. 退出重进后状态不丢
