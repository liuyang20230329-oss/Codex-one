## 1. 产品定位

`37°` 当前是一套“社交 App 第一阶段可联调版本”。

当前目标不是完成全部社交链路，而是打通以下主线:

1. 手机号主链登录注册
2. 账号资料与认证闭环
3. 文字聊天 MVP
4. 可运行的本地 API 和 WebSocket 联调环境
5. 广场、圈子、消息、我的四个模块的移动端页面壳子

## 2. 当前运行模式

App 当前支持 3 种运行模式:

| 模式 | 说明 | 当前状态 |
| --- | --- | --- |
| `localApi` | 默认模式，优先连接本地 API | 已启用 |
| `demo` | 当本地 API 不可达时回退到本地演示仓库 | 已启用 |
| `firebaseLegacy` | 兼容旧 Firebase 登录链路 | 已保留，但不是主链 |

说明:

- 默认编译配置会优先走 `localApi`
- 如果 `LOCAL_API_BASE_URL` 无法访问，会自动回退到 `demo`
- `firebaseLegacy` 仅作兼容保留，不是现阶段主认证方案

## 4. 登录注册与认证

### 4.1 登录

当前需求实现:

- 使用手机号 + 密码登录
- 登录后进入四 tab 主页面
- 登录状态由 `AuthController` 管理
- 本地 API 模式下，成功登录后保存 token 和用户信息

当前限制:

- 不支持邮箱主链登录
- 不支持验证码登录

### 4.2 注册

当前需求实现:

- 使用昵称 + 手机号 + 密码注册
- 注册页包含二次密码确认
- 注册页包含“同意协议”勾选
- 本地 API 模式下，注册前自动发送一条本地调试短信验证码，再完成注册

当前限制:

- 没有独立的协议详情页
- 验证码仍使用本地固定调试码，不是真实短信平台

### 4.3 社会化登录

当前需求实现:

- 登录页展示“微信登录”“QQ 登录”按钮
- 后端有 `/auth/social/wechat`、`/auth/social/qq`、`/auth/social/bind`、`/auth/social/unbind`

当前限制:

- 当前点击会收到“待配置”提示
- 未接入真实 SDK、开放平台密钥和回调流程

### 4.4 手机号认证

当前需求实现:

- 账号中心可发起手机号认证
- 界面会展示演示验证码，便于完整验证流程
- 手机号认证成功后可解锁正式私聊能力

当前规则:

- 未完成手机号认证时:
  - 可浏览主页面
  - 可进入系统引导会话
  - 不可发起正常私聊

### 4.5 身份证认证

当前需求实现:

- 账号中心可提交真实姓名与身份证号
- 提交后立即进入已通过状态
- 后端已记录实名审核申请和审计日志

当前限制:

- 暂未接 OCR
- 暂未接真实人工审核流
- 当前本地 API 为“开发自动通过”

### 4.6 本人头像认证

当前需求实现:

- 账号中心可发起本人头像认证
- 完成后记录相似度分数
- 更换头像会自动重置本人头像认证状态

当前限制:

- 暂未接真实活体或人脸 SDK
- 当前本地 API 为“开发自动通过”

## 5. 首页与视觉主题

### 5.1 首页壳子

当前需求实现:

- 四个 tab: 广场、圈子、消息、我的
- 顶部主题头图展示当前模块说明
- 底部导航使用圆角浮层容器
- 页面整体视觉从默认白底列表升级为品牌化舞台布局

### 5.2 男女主题

当前需求实现:

- 男性主题偏深沉、黑灰、冷蓝高亮
- 女性主题偏温暖、粉雾、柔和高亮
- 主题色会影响首页、聊天、我的等主要模块卡片和高亮状态

当前限制:

- 登录页仍为统一品牌深色风格，没有按性别切换
- 广场、圈子人物卡仍为示意式 UI，不是真实头像与素材

## 6. 广场

当前移动端需求实现:

- 顶部 Banner 轮播
- 平台系统通知卡
- 地区 / 年龄 / 性别筛选
- 用户推荐卡片列表
- 用户卡片展示性别、年龄、地区、距离、签名、标签、认证状态、在线状态

当前数据来源:

- 移动端广场页面当前使用本地静态数据
- 不直接读取 `local-api /square/*`

后端当前能力:

- Banner 接口
- 系统通知接口
- 用户筛选接口
- 保存筛选条件接口

当前差距:

- 前端尚未接通广场接口
- 没有用户详情页跳转
- 没有真实搜索页

## 7. 圈子

当前移动端需求实现:

- 圈子动态列表
- 悬浮“发布动态”按钮
- 全屏发布页
- 发布页支持:
  - 文案
  - 地址选择
  - 图片选择
  - 语音选择
  - 作品选择
  - 草稿保留

当前发布规则:

- 至少有一项内容才能发布
- 发布成功后，新动态会插入当前本地列表顶部

当前数据来源:

- 圈子列表和发布结果当前仍在移动端本地内存中维护
- 不直接写入 `local-api /circle/*`

后端当前能力:

- 动态列表
- 动态详情
- 发布动态
- 编辑动态
- 评论
- 举报

当前差距:

- 移动端尚未接通圈子 API
- 没有真实评论列表页
- 没有真实动态详情页

## 8. 消息与聊天

### 8.1 会话列表

当前需求实现:

- 关系分组:
  - 好友
  - 热聊
  - 关注我的
  - 我关注的
- 会话列表
- 未读红点
- 新建会话
- 置顶会话
- 删除会话
- 全部已读

### 8.2 聊天详情

当前需求实现:

- 文本消息发送
- 表情、图片、语音、视频快捷发送入口
- 发送权限控制
- 草稿保留
- 已送达 / 已读 / 发送中 / 失败状态文案展示

说明:

- 图片、语音、视频当前在聊天内仍为占位消息，不是真实媒体上传链路

### 8.3 聊天规则

当前需求实现:

- 系统引导会话不受手机号认证限制
- 普通私聊需要先完成手机号认证
- 会话按“置顶 > 最后活跃时间”排序
- 后端收到消息后会生成演示自动回复

### 8.4 WebSocket

当前需求实现:

- 连接 `/ws/chat`
- 收到事件后刷新会话和消息列表
- 当前支持的事件类型包括:
  - `connected`
  - `conversation-created`
  - `conversation-updated`
  - `conversation-read`
  - `conversation-read-all`
  - `conversation-deleted`
  - `message-created`

## 9. 我的与账号中心

当前需求实现:

- 个人 Hero 卡片
- 资料完成度
- 认证完成度
- 基础资料编辑:
  - 昵称
  - 头像主题
  - 性别
  - 出生年月
  - 地区
  - 个性签名
- 视频介绍编辑
- 作品新增与删除
- 认证中心:
  - 手机号认证
  - 身份证认证
  - 本人头像认证
- 退出登录

当前限制:

- “基础设置”卡片当前主要为本地 UI 状态切换，没有真正持久化到后端
- 设备管理、黑名单、通知设置、注销账号虽然后端有接口，但移动端暂无完整页面入口
