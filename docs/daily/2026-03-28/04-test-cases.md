## 2. 资料与认证测试用例

### TC-ACCOUNT-001 编辑昵称与头像主题

- 前置条件: 用户已登录
- 操作步骤:
  1. 打开账号中心
  2. 点击编辑资料
  3. 修改昵称
  4. 切换头像主题
  5. 保存
- 预期结果:
  - 页面展示新的昵称
  - 页面展示新的头像主题
  - 如果之前已完成人脸认证，则认证状态被自动重置

### TC-ACCOUNT-002 手机号认证成功

- 前置条件: 用户已登录
- 操作步骤:
  1. 进入手机号认证
  2. 输入手机号
  3. 获取验证码
  4. 输入页面展示的 demo 验证码
  5. 提交确认
- 预期结果:
  - 手机号状态变为 `Verified`
  - 手机号以脱敏格式展示

### TC-ACCOUNT-003 实名认证成功

- 前置条件: 用户已登录
- 操作步骤:
  1. 进入实名认证
  2. 输入真实姓名
  3. 输入 18 位身份证号
  4. 提交
- 预期结果:
  - 实名状态变为 `Verified`
  - 身份证号以脱敏格式展示

### TC-ACCOUNT-004 未实名时禁止本人头像认证

- 前置条件: 用户已登录但未完成实名
- 操作步骤:
  1. 尝试直接进行本人头像/人脸认证
- 预期结果:
  - 无法完成认证
  - 系统提示需先完成实名认证

### TC-ACCOUNT-005 本人头像认证成功

- 前置条件:
  - 用户已登录
  - 已完成实名认证
- 操作步骤:
  1. 进入本人头像认证
  2. 勾选确认当前头像属于本人
  3. 点击开始认证
- 预期结果:
  - 人脸认证状态变为 `Verified`
  - 头像出现认证标识

### TC-ACCOUNT-006 重启后账号状态保留

- 前置条件:
  - 已完成部分账号资料或认证步骤
- 操作步骤:
  1. 关闭 App
  2. 重新打开 App
- 预期结果:
  - 已登录 Demo 账号仍保持登录
  - 昵称、头像主题、手机号认证状态等本地状态仍保留

## 3. 文字聊天 MVP 测试用例

- 操作步骤:
  1. 进入 Chats 页面
- 预期结果:

### TC-CHAT-002 打开会话详情

- 前置条件: 会话列表已有数据
- 操作步骤:
  1. 点击任意会话
- 预期结果:
  - 进入消息详情页
  - 历史消息正确展示

### TC-CHAT-003 发送文字消息

- 前置条件: 已进入某个会话
- 操作步骤:
  1. 输入文字内容
  2. 点击发送
- 预期结果:
  - 新消息追加到消息列表底部
  - 会话最后一条消息同步更新

### TC-CHAT-004 未完成认证也可聊天

- 前置条件: 用户已登录但未完成全部认证
- 操作步骤:
  1. 直接进入 Chats 页面
  2. 打开发消息
- 预期结果:

  - 可以正常发送消息

- 操作步骤:

  2. 或完成手机号 / 实名 / 人脸认证任一步
  3. 返回聊天列表
- 预期结果:
  - Concierge 会话产生新的系统消息
  - 会话排序和未读数同步更新

### TC-CHAT-006 重启后聊天记录保留

- 前置条件: 已发送过聊天消息
- 操作步骤:
  1. 关闭 App
  2. 重新打开 App
  3. 回到原会话
- 预期结果:
  - 历史消息仍存在
  - 会话最后一条消息与未读状态保持一致
