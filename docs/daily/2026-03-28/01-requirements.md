# Codex One 需求文档

- 文档版本: `2026.03.28-r1`
- App 版本: `0.2.0-dev.1+20260327`
- 分支: `2026.3.27`
- 日期: `2026-03-28`
- 状态: `Draft`

## 1. 产品定位

Codex One 是一个面向移动端的社交 App，一期目标是先把账号体系跑通，再逐步扩展到文字聊天、语音房、视频通话和用户资料体系。

当前仓库已经完成一个可安装、可进入、可测试的认证起步版，适合作为后续社交功能的基础工程。

## 2. 今日已完成内容

### 2.1 已完成的用户能力

- 用户可在登录页和注册页之间切换
- 用户可使用邮箱和密码完成注册
- 用户可使用邮箱和密码完成登录
- 用户登录后可进入首页
- 用户可主动退出登录
- App 在未配置真实 Firebase 时会自动切换到 Demo 模式，便于本地测试

### 2.2 已完成的工程能力

- Flutter 移动端工程已建立
- Android 工程已生成，可输出 APK
- Firebase Auth 启动逻辑已接入，支持真实 Firebase 与 Demo 双模式
- 登录注册表单校验、异常提示、认证状态管理已完成
- 基础测试与静态检查已跑通

## 3. 当前版本范围

### 3.1 P0 范围

本阶段目标是让“注册/登录/退出/进入首页”可以稳定测试。

P0 具体范围如下：

- 邮箱注册
- 邮箱登录
- 退出登录
- 登录态切换
- 认证失败错误提示
- 登录后首页展示基础用户信息
- 为后续文字、语音、视频功能预留入口位

### 3.2 P0 验收标准

- 新用户可以成功注册
- 已注册用户可以成功登录
- 密码错误时要有明确提示
- 登录成功后首页能展示用户昵称、邮箱、用户 ID
- 退出登录后应返回登录页
- 未配置真实 Firebase 时，Demo 账号可直接完成流程测试
- Android 端能够安装并启动 debug APK

## 4. 下一阶段需求拆分

### 4.1 P1 用户资料

- 用户补充昵称
- 用户上传头像
- 用户编辑个性签名
- 用户查看个人资料页

### 4.2 P1 文字社交

- 会话列表
- 单聊
- 群聊
- 消息发送状态
- 未读消息数

### 4.3 P2 语音社交

- 语音房创建
- 语音房加入/退出
- 麦位管理
- 麦克风状态管理

### 4.4 P2 视频社交

- 一对一视频通话
- 多人视频房间
- 通话邀请
- 通话状态管理

## 5. 技术方案约束

- 客户端当前采用 `Flutter`
- 认证方案当前采用 `Firebase Authentication`
- 在真实 Firebase 配置缺失时，使用本地 Demo 认证作为降级方案
- 本阶段不自建密码存储逻辑，不在业务数据库保存明文密码或密码哈希
- Android 为当前优先测试平台

## 6. 当前已知依赖与风险

- 真实 Firebase 项目尚未配置完成，目前默认仍可回退到 Demo 模式
- 文字聊天、语音、视频仅完成产品入口预留，尚未进入业务开发
- 用户资料、消息系统、关系链、实时音视频服务尚未接入后端

## 7. 今天的结论

今天这版可以定义为：

`认证起步版 / Auth Starter`

它已经满足“你可以在手机上安装并测试登录注册流程”的目标，接下来最合理的推进顺序是：

1. 接通真实 Firebase 登录注册
2. 增加用户资料页
3. 开始做会话列表和文字聊天
