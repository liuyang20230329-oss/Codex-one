## 1. 今晚完成的范围

### 1.0 第二批联调底座
- 新增 `local-api` SQLite 本地服务端，统一挂载到 `/api/v1/*`
- 新增 `auth / users / chat / circle / square / notifications / reviews / admin / search` 路由组
- 新增 `ws://127.0.0.1:3001/ws/chat`，用于聊天实时刷新
- 新增烟雾测试脚本 `local-api/scripts/smoke-test.js`
- 兼容遗留 `.env` 中的旧端口 `3000` 和旧库 `37degrees.db`，自动切到隔离的 `3001 + 37degrees-v2.db`

### 1.1 认证主链切换为手机号优先
- 登录与注册入口改为手机号 + 密码
- 新增短信验证码申请、手机号验证、密码重置占位流程
- 新增微信 / QQ 第三方登录入口和后端占位接口
- App 启动优先尝试 `local-api`，失败时自动回退到 Demo 数据

### 1.2 消息与聊天全量增强
- 新增会话创建、置顶、删除、全部已读
- 修复系统向导会话在 `local-api` 模式下的权限判断，兼容 `concierge-*` 会话 ID
- 聊天页新增快捷发送入口：表情 / 图片 / 语音 / 视频占位消息
- 消息气泡新增消息类型标签与媒体说明字段展示
- 会话卡片新增在线状态、置顶状态和操作菜单

### 1.3 后端数据覆盖范围
- 广场接口支持 Banner、系统通知、用户筛选、筛选条件保存
- 圈子接口支持动态列表、动态详情、发布、编辑、评论、举报
- 用户接口支持完整资料、作品、隐私设置、设备会话、黑名单、注销
- 后台接口和审核接口已具备基础 RBAC / 审核 / 日志落点

### 1.4 品牌统一
- 新增品牌常量与品牌锁定组件
- 应用标题统一为 `37°`
- Android 安装名更新为 `37°`
- Windows 窗口标题和产品元数据更新为 `37°`
- 登录页、首页、消息引导文案统一为 `37°` 品牌表达

### 1.5 账号体系闭环增强

- 新增资料完成度计算逻辑
- “我的”页顶部新增资料完成度展示

- 未完成认证的私聊会话增加 `待认证` 提示

### 1.7 广场与圈子体验增强

## 4. 风险与后续

- 当前注册仍以邮箱密码为基础，手机号认证属于账号信任闭环的一部分；后续如需切到“纯手机号注册”，需要补充新的认证后端和登录入口
- 当前圈子媒体、作品、头像与视频仍是结构化演示数据，后续可接真实上传、审核和存储服务
- Android 包名和可执行文件名暂未整体迁移到 `com.thirtysevendegrees.app`，本轮先优先统一用户可见品牌
