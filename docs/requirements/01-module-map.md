# Codex One 模块划分

## 一、模块总览

当前建议把产品拆成 6 个一级模块。

1. 账号与认证
2. 广场
3. 圈子
4. 消息与聊天
5. 我的与设置
6. 公共能力

## 二、模块边界

### 1. 账号与认证

负责用户进入 App 前后的身份体系。

包含:
- 登录
- 注册
- 手机号认证
- 身份证实名
- 本人头像和人脸认证
- 账号状态和认证状态

不包含:
- 聊天消息
- 广场推荐
- 圈子动态

### 2. 广场

负责陌生人发现和推荐曝光。

包含:
- 顶部 Banner
- 平台通知
- 用户列表
- 地区、年龄、性别等筛选

不包含:
- 私聊详情
- 朋友圈发布

### 3. 圈子

负责附近的人动态流和发布能力。

包含:
- 动态列表
- 发布按钮
- 发布页
- 文案、定位、图片、语音、动图、网址

不包含:
- 私聊
- 账号设置

### 4. 消息与聊天

负责关系分组、会话列表、聊天详情。

包含:
- 好友
- 热聊的
- 关注我的
- 我关注的
- 会话消息列表
- 红点和未读
- 单聊详情

不包含:
- 用户认证
- 广场筛选

### 5. 我的与设置

负责用户个人展示、资料完善、基础设置。

包含:
- 头像
- 性别
- 年龄
- 出生年月
- 个性签名
- 视频介绍
- 我的作品
- 基础设置

不包含:
- 消息分发
- 推荐算法

### 6. 公共能力

负责所有模块共用的底层能力。

包含:
- 底部导航
- 页面跳转
- 上传能力
- 权限申请
- 审核状态
- 系统通知
- 埋点
- 缓存和本地存储

## 三、推荐开发顺序

今晚最适合继续推进的顺序:

1. 账号与认证闭环
2. 我的与设置
3. 消息与聊天 MVP
4. 广场基础版
5. 圈子基础版
6. 公共能力补齐

## 四、你后续给我任务时的写法

建议按下面格式发任务:

```md
模块: 消息与聊天
页面: 会话列表
目标: 让用户快速看到最新聊天和未读红点
必须完成:
- 顶部分组入口
- 会话列表
- 未读红点
- 点击进入聊天页
可以延后:
- 搜索
- 置顶
- 删除会话
参考文档:
- docs/requirements/modules/40-messages-and-chat.md
```
