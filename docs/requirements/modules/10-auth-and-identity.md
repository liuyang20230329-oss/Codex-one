# 模块一：账号与认证

## 1. 模块目标

这个模块负责把“用户是谁、是否可信、是否能继续使用功能”定义清楚。

## 2. 当前建议包含的页面

1. 登录页
2. 注册页
3. 认证中心页
4. 手机号认证页
5. 身份证实名页
6. 本人头像和人脸认证页

## 4. 核心业务规则

这里建议你重点补下面这些规则。

- 未认证用户是否允许聊天:
- 未手机号认证用户是否允许发消息:
- 未实名用户是否允许发布动态:
- 未本人认证用户是否限制曝光:
- 认证中的用户是否能继续使用:

## 5. 用户状态流转

建议你按这条链路补充:

```text
注册 -> 登录 -> 进入认证中心 -> 手机号认证 -> 实名认证 -> 本人头像/人脸认证 -> 完成可信用户状态
```

## 6. 需要你线下补充的重点

- 认证顺序是否固定
- 是否允许跳过
- 审核是否人工介入
- 失败后是否允许无限次重试
- 认证完成后是否要给资料页徽章
